1. **SQLSTATE 42501 (run 29983887580 root cause)** — the preflight pinned

   unqualified objects in the FIRST schema of search_path, so sqlx's
   unqualified `CREATE TABLE _sqlx_migrations` targeted the system catalog.
2. **enrollment.rs** still used the pre-ADR-0006 per-test SCHEMA pattern
   (after_connect search_path pin) — the canonical runner migrates `public`,
   queries ran in an empty schema → 500s. Converted to the throwaway

3. **Amendment 1 correction** — my first fix used `public, pg_catalog,
   pg_temp`, which Amendment 1 explicitly REJECTS (public ahead of the
   catalog lets public objects shadow built-ins). Final value:
   **`public, pg_temp`** — pg_catalog unnamed (implicit-first lookup),
   public first explicit (creation target), pg_temp last (hardening).
   Evidence: `canonical_migrations_create_public_history_with_catalog_and_temp_precedence`
   (all three pairwise lookup edges, relations AND types, real PG16).
   ADR-0006 §1 body text on main was stale vs Amendment 1; advisor
   reconciled it in PR #44 (main `8a07668`).
4. **Sol review finding 1: preflight TOCTOU** — preflight ran before
   run_direct acquired sqlx's advisory lock. `migrate_with_bounds` now takes

   Evidence: `canonical_migration_preflight_runs_under_migration_lock`
