## Conventions (carried, updated)

- Infra-needing tests are `#[ignore = "requires ...; CI <job> runs it"]`
  and run in a dedicated CI job with `--include-ignored`. Default
  `cargo test --workspace` must stay infra-free. Missing infra = loud
  failure, never a skip (§13, scope rule 6).
- DB tests share one database per job: isolate by fresh random UUIDs per
  case, NEVER by TRUNCATE (cross-test race). Package-byte fixtures must be
  unique per device when assertions compare bytes across devices.
- Stack branches sequentially; keep each PR focused and ≲600 hand-written
  lines. One commit per branch; amend + `--force-with-lease` on restack.
- Issue numbering: Opus 001/002/005/006, K3 003/004/007. ADR numbering:
  Opus 0001/0002, K3 0003. **Next free: issue 008, ADR 0004.**
- sqlx notes: runtime `Migrator::new(&Path)` (feature "migrate") — the
  `migrate!` macro is NOT used; bind `Vec<Vec<u8>>` as `bytea[]` via
  UNNEST; `to_jsonb(t)::text` for generic row dumps. Cargo.lock records ALL
  optional deps (incl. unbuilt backends like sqlx-mysql) — plain
  `cargo audit` scans them; justify ignores in `.cargo/audit.toml`.
- Canary scan never places canaries in paths/query strings or in fields
  meant to be stored — bodies and headers of rejected/anonymous requests
  only (pinned by `injection_points_keep_canaries_out_of_urls`).
- Spike/evidence branches are named `k3/spike-*`, based on whatever tree
  they need, and are never merge candidates.
- CI time budget: all jobs 15-min timeout; current wall ~6-7 min per run.
