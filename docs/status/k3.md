## SESSION 2 (2026-07-22) — blocking-review fixes on #39

Blocking review landed three findings; fixed two, held one:

1. **Founding race (fixed, `c140dba`)** — founder status is now atomic with
   groups-row creation: for a Welcome the upsert IS the founder test
   (`rows_affected == 1` → founder; `0` → fall through to
   `is_participant_in`, which a racing non-member fails after Postgres
   unblocks it on the winner's committed row). Proof:
   `founding_race_admits_only_winning_founder` (16-round barrier race, real
   PG16 in CI).
2. **Welcome at-least-once wrt consumption (fixed, `c140dba`)** — the
   gateway now marks NOTHING at push time; a welcome is marked delivered
   only by the client's post-verification Subscribe naming its group (F2's
   consumption signal). Re-pushed on every connect until then.
   Proof: `welcome_redelivered_until_subscribe` (disconnect-before-consume →
   redelivery; subscribe → no further re-push).
3. **Migration architecture (HELD)** — ADR-0006 (Sol) decides one canonical
   migrator vs governed partial migrators. The `ignore_missing` + 0004
   numbering setup stays as-is until charge accepts that ADR; #39 stays
   DRAFT until then and merges after.

Note: `migrations/0004` still carries the old push-then-mark comment —
deliberately not edited (changing an applied migration's text changes its
sqlx checksum and breaks persistent dev DBs); authoritative behavior is in
`gateway.rs`/`store.rs` comments. Docker is unavailable on this machine, so
